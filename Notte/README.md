# Notte

**面向 Apple 生态的轻量级结构化知识工作台**

---

## 这是什么

Notte 是一款专为 iPhone 设计的笔记与内容组织应用，用 SwiftUI + SwiftData + CloudKit 构建。

它不是另一个通用笔记工具。Notte 的出发点是：大多数知识管理产品要么太重，要么太散——用户在"快速记录"和"长期整理"之间始终面临断裂。Notte 试图在这两端之间建立一座桥。

核心目标只有一个：

> **让用户能够快速记录、自然形成结构、逐步整理内容、长期沉淀知识。**

---

## 核心模型

Notte 所有功能都建立在一个统一的层级结构之上：

```
Collection
  └── Page
        └── Node（可无限嵌套）
              └── Block（内部模型，用户不可见）
```

**Collection** 是内容的组织空间，可以是一个项目、一门课程、一个研究专题。

**Page** 是完整的内容页面，由多个 Node 组成。

**Node** 是 Notte 的核心单位——每个 Node 是一个可嵌套的大纲条目，拥有标题和内容区。Node 的嵌套深度同时决定其视觉层级，无需额外设置类型或级别。

> **Block** 是 Node 内容区的底层存储单元（文字段落、图片、代码块等），属于开发者层面的内部模型，不作为概念暴露给用户。

---

## 存储策略

Notte 采用 **local-first** 架构：

- 所有内容优先存储在本地，App 完全可以离线使用
- iCloud 作为轻量同步层，负责跨设备的核心数据同步
- 附件与大体积文件默认保留在本地，不占用 iCloud 主存储空间

这意味着即使没有订阅大容量 iCloud，Notte 的核心能力也完整可用。

---

## MVP 范围

当前版本（MVP）聚焦于验证核心主路径的可用性：

- 创建与管理 Collection
- 在 Collection 中创建和浏览 Page
- 在 Page 中通过 Node 大纲组织文字内容
- Node 的创建、编辑、层级调整与排序
- 数据在重启后完整保留
- 基础 iCloud 同步

模板、思维导图视图、跨页面 Node 迁移、内容引用与复用、协作等能力将在 MVP 之后的阶段陆续推出。

## 设计理念

Notte 认为，笔记工具的护城河不是功能数量，而是使用感受：

- 打开就想记
- 记完很容易整理
- 整理后很容易复用
- 长期积累不会乱

如果这条链路被打通，Notte 就有机会成为某类用户真正离不开的内容空间。
